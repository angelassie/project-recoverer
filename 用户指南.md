# User Guide — Project Recovery Assistant

## What Is This Tool?

This is a **Project Recovery Assistant**. It helps you pick up where a Codex conversation left off after an error or interruption.

It does three things:

1. **Recovers your project context** — reads the Memory (记忆体) files to understand what was being worked on
2. **Analyzes error messages** — if your conversation was interrupted by an error, paste the error text and it will categorize and explain what went wrong
3. **Shows unfinished tasks** — tells you exactly what to do next

---

## Quick Start

### Method 1: Run from this project folder

Open PowerShell, navigate to this folder, and run:

`powershell
python core\recoverer.py
`

This will scan the current project, load the Memory files, and show you the recovery report.

### Method 2: Point to any project folder

`powershell
python core\recoverer.py "F:\codex\YourProjectFolder"
`

### Method 3: With an error message

If your Codex conversation was interrupted by an error, copy the error text and pass it:

`powershell
python core\recoverer.py "F:\codex\YourProjectFolder" "The error text goes here"
`

### Method 4: Initialize memory for a new project

For a brand-new project, generate the Memory folder and files:

`powershell
python core\recoverer.py "F:\codex\NewProject" --init-memory
`

---

## How It Works

When you run the assistant, it follows these steps:

| Step | What Happens |
|------|-------------|
| 0 | Checks if Memory (记忆体) exists; creates it from templates if missing |
| 1 | Loads all Memory files (Index, Status, Conventions, Dialogue Logs) |
| 2 | Analyzes any error text you provided |
| 3 | Scans project files to understand the codebase |
| 4 | Finds incomplete tasks from the Status file |
| 5 | Suggests the next action |
| 6 | Saves a recovery report to outputs/ folder |

---

## The Memory Folder (记忆体)

Every project has a 记忆体 folder with four files:

| File | Purpose |
|------|---------|
| 索引.md | Project overview and file index |
| 状态.md | Task checklist — what's done, what's pending |
| 约定.md | Development rules and conventions for this project |
| 对话记录.md | Chronological log of Codex conversations |

You can edit these files manually. The assistant reads them to understand your project state.

---

## Tips

- **After a Codex interruption**: Copy the error message, run the assistant with it, then paste the recovery report back into Codex to resume.
- **Keep 状态.md updated**: Mark tasks as done with [x] so the assistant knows what's left.
- **New projects**: Always run --init-memory first to set up the Memory folder.
- **Check outputs/**: Every recovery generates a JSON report you can review later.

---

## Troubleshooting

| Problem | Solution |
|---------|----------|
| "Memory not found" | Run with --init-memory to create it |
| No tasks shown | Check that 状态.md has - [ ] task entries |
| Error analysis empty | Make sure the error text is passed as the second argument |
| Python not found | Install Python 3.8+ or use python3 instead of python |
