# Project Memory Conventions

## Project Path
- Project code directory: F:/codex/Project002_CodexMonitor/
- All code stored in core/ directory
- Output files stored in outputs/ subdirectory

## Memory Structure
Each new project root directory contains a memory/ folder with:

| File | Purpose |
|------|---------|
| memory/index.md | Overview of all files, quick navigation |
| memory/dialogue_logs.md | Chronological record of key discussions, decisions, and progress |
| memory/status.md | Current project status, completed/in-progress/todo items |
| memory/conventions.md | Tech selection, coding standards, memory usage conventions |

## Development Standards

### File Creation Method
- Prefer shell_command with Out-File -Encoding UTF8 for creating text files
- Avoid using apply_patch for files containing Chinese or special characters
- Reason: Unescaped Chinese characters in patch format can break outer JSON structure

### Common Errors Recorded

#### JSON Parse Error
- Trigger: Using apply_patch to create/modify files with Chinese content
- Root Cause: Chinese characters in patch content broke JSON structure
- Solution: Use shell_command + Out-File instead

## Project Tech Stack
- Python 3.x
- pathlib for path management
- json module for reading/writing session data
- Codex thread management tools (create_thread, send_message_to_thread, read_thread, etc.)
- Regular expressions for extracting error messages

## Conversation Conventions
- At the start of each new conversation, read files under memory/ to restore context
- Actively update memory files during conversations
- Confirm memory status is up-to-date before ending conversations

## Core Workflow

### When a conversation is interrupted, the recovery assistant steps are:
1. **Read Memory** -- Get previous project status and context
2. **Analyze Errors** -- Extract error info from the interruption
3. **Reconstruct Context** -- Recover lost information from memory and project files
4. **Generate Recovery Report** -- Tell user where things left off and what to do next
5. **Update Memory** -- Mark recovery as complete
